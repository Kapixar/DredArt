DredArt Help section

1. Overview

In this section we will focus on Creator - the UI overlay.

In top bar there are 3 new buttons:

  • Pallete - open / close Creator
  • Red Cross - remove Holo (removes textures)
  • Refresh - Refreshes textures in case of their absence.

In Creator menu you can go to Render, where you can create new pixel art. Help for it is build-in.

After you made Pixel Art, to begin painting follow these steps:

  1. Upload Pixel Map created to Use pixel art button
  2. Summary will show up. If your pixel art is not in default ship's corner, enter correct coordinates.
  3. Click Generate!
  4. First time warning will show up. Read it!
  5. New tabs showed up:
    • Map - mini map of the pixel art. Hovering on pixels will show their coordinates.
    • Holo - Holo uses textures to display big stencils of the pixel art.
    • Settings - set your preferences

2. Tool Tabs

Map

Very simple tab that contains map of pixel art.

Scroll to move around

To scroll horizontaly without clicking use SHIFT+SCROLL

Double click color to hide other colors. Double click again to show all.

Holo

Overview of elements:

  1. Help messages
  2. Coordinates for blocks you need to place to display hologram. They depend on coords you entered before.
  3. Floating panel
    • Show all - display map of colors as Holo. Correctly placed paint will not be displayed.
    • Find errors - use to check for errors in placing paint. Correct pixels should turn into one shade of grey.
    • Color from coords - After enterring a coordinate returns Color.
    • Corner Selector - select ship sector in which you want to show Holo (exist due to limitations)
    • Quick help button
  4. Sort panel - Sort the colors below.
  5. Individual colors - clicking on them displays a stencil where to put paint. Black - don't put / transparent - put. Values in corners represent the amount of this color in chosen part of the ship.

Limitations

  • Curently there is a size limit for texture, caused by cogg's fights with WebGLContext. For us that means that Holo won't always fit into the Texture Atlas. That's the reason Holo have to be splited. Solution is to use holo in empty ship, with max zoom.
  • Various bugs may have been solved in Troubleshooting part
  • In any moment cogg can make this stop working (pls no i worked on it for too long)

Settings

Set up your preferences

    Possible options:
  • Change your texture pack that will be insterted when you click Disable Holo. It needs to be fully functional Mod (/img/ folder inside).
  • Change blocks that emit Holo. Five available options.
  • Change color of Holo's stencil if you need.
  • Customize the Show all font
  • Enable text in the Find errors
  • Always show every color in Holo tab, independently of chosen quarter.
  • Specify when Holo breaks into smaller parts.

3. Troubleshooting

  • Image contains colors that don't exist in Dredark color pallete.

    Make sure you used correct image. It's possible that you dragged scaled image from Discord, first open full version in broswer.

  • Grey screen

    Happens after combo of rendering approx. 80 colors. Refresh site to fix.

  • No visible holo after clicking on color

    Use holo in empty ship, fly to the Holo generating block and max zoom-in. If doesn't help, refresh site and repeat.

  • No paint codes in paint's R menu

    Atlas limitations. Try refreshing Holo, zooming, etc, or try remember the pallete.

  • No paint cursor

    Try refreshing Holo.

  • Holo is wrong with what is on the Map

    Make sure you entered correct image. Too many people do that.

  • There is a thing not described here

    Report it in MOSIAC Discord server.

4. Tips and Tricks

  • To make painting faster:
    • use gravity to move paint around ship
    • don't fix mistakes right away, use Find errors when you finish whole
    • start from the largest area colors, then use Show all
    • gather all the paint you need before you start
  • you can remove the DredArt icon for extension bar to make more space
  • experiment with Render settings
